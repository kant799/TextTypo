## 页面设计
### 1.页面蓝图
请严格遵循以下内容框架，这是不可协商的。每一部分都有其明确的功能，共同构成页面的节奏感。

#### 主内容区（Main Body）：页面的核心，必须采用 8+4 的非对称网格布局。
**海报展示区域 (8列)**
	此区域呈现用于**海报设计**，创建一个3:4比例的容器，所有海报相关设计都会在这个区域完美呈现。

**海报分享(Download):**
- 在**设计说明区域**添加下载按钮，用于将**海报展示区域**区域的呈现内容内容，以图片形式进行下载。(下载功能使用**snapDOM**模块，文件名格式：[海报主题名称]-[日期].png )
### 2. 设计基因
#### 视觉元素:
- 色彩：使用用户指定的颜色信息，运用对比色来制造视觉冲击力，或运用流行色来增加时尚感。
- 背景：根据海报整体色彩科学，为背景添加符合海报风格的色彩渐变。
- 文案：
	- 不随意改变或增添用户提供的文案内容
	- 英文字体偏大、字重加粗，提升设计格调。
	- **禁止**添加任何emoji和图标元素，使用最原汁原味的色彩和文字变化来完成设计创作。
#### 页面布局
- 视觉协调：禁止定义 body 标签的margin、border、padding对应数值。同时为 body 标签或最外层容器设置 height: 100vh; 和 overflow: hidden。
- 展示为主：禁止所有与鼠标hover状态有关的动态变化。
- 响应式：海报会在所有尺寸的屏幕中完整呈现，无需滚动查看。
### 3. 技术规格交付物: 
单一、自包含的 HTML5 文件
- 技术栈: 使用 TailwindCSS、Google Fonts、和 snapDOM，均通过 CDN 引入。
	- snapDOM 模块解释：
	```
	<!-- CDN 引入 -->
	<script src="https://cdn.jsdelivr.net/npm/@zumer/snapdom/dist/snapdom.min.js"></script>
	<!-- 函数调用 -->
	<script>
		// 使用 snapDOM 下载卡片
		const el = document.querySelector('#target');  
		const result = await snapdom(el, { scale: 2 });
		const img = await result.toPng();  
		document.body.appendChild(img);
		await result.download({ format: 'jpg', filename: 'my-capture' });
	</script>
	```
- 内容: 不得省略用户输入内容的核心要点。
- 适配: 优先适配 1200x1600 窗口的宽高比，并确保响应式布局。
